# Paw — Service & Feature Specification

## 1. Service Overview

### What is Paw?

Paw는 사람과 AI 에이전트가 동일한 대화 공간에서 자연스럽게 소통하는 **AI-네이티브 메신저**입니다.

기존 메신저에서 AI를 사용하려면 별도의 봇 채팅방이나 외부 앱으로 이동해야 합니다. Paw에서는 에이전트가 사용자와 동등한 대화 참여자입니다. 그룹 대화에 에이전트를 초대하면, 에이전트는 맥락을 이해하고 실시간으로 응답하며, 필요한 도구를 호출합니다.

### Target Users

| Persona | Description |
|---------|-------------|
| 개인 사용자 | 보안 메신저와 AI 비서를 하나의 앱에서 사용하고 싶은 사람 |
| 팀/조직 | 업무 협업에 AI 에이전트를 자연스럽게 통합하려는 그룹 |
| 에이전트 개발자 | 자신의 AI 에이전트를 마켓플레이스에 등록·배포하려는 빌더 |

### Core Value Proposition

1. **에이전트가 대화의 일원** — 별도 UI나 명령어 없이, 채팅방에 에이전트를 초대하면 대화에 참여
2. **E2EE + 에이전트 참여** — 종단간 암호화된 대화에서도 에이전트가 안전하게 동작하는 구조
3. **실시간 스트리밍** — 에이전트 응답이 토큰 단위로 실시간 표시되고, 도구 호출 상태를 추적
4. **마켓플레이스** — 에이전트 탐색·설치·권한 관리를 앱 안에서 완결

### Competitive Positioning

| Service | Paw와의 차이 |
|---------|-------------|
| **Telegram / KakaoTalk** | 일반 메신저. AI 봇이 있지만 명령어 기반이며 대화의 맥락을 이해하지 못함 |
| **Slack / Discord** | 팀 협업 도구. 봇/앱이 있지만 별도 설치와 설정이 필요하고 자연어 대화가 아닌 슬래시 명령 중심 |
| **ChatGPT / Claude** | AI 대화 전용. 메신저가 아니라 사람 간 대화, 그룹 채팅, 푸시 알림 등이 없음 |
| **Signal** | 보안 메신저. E2EE에 집중하지만 AI 에이전트 통합이 없음 |

**Paw는 "사람 간 메신저"와 "AI 에이전트 인터페이스"를 하나로 통합합니다.**

## 3. Features

### 3.1 인증

전화번호 기반 OTP 인증. 비밀번호는 없습니다.

- 전화번호 입력 (국가코드 포함, 한국 번호 자동 보정)
- SMS OTP 6자리 인증 (60초 후 재전송 가능)
- 디바이스 등록 (기기 이름 지정, 보안 키 자동 생성)
- 프로필 설정 — username (3-20자), 전화번호 검색 허용 토글. 건너뛰기 가능

### 3.2 1:1 및 그룹 대화

사람 간 1:1 대화와 그룹 대화를 지원합니다.

**대화에서 확인 가능한 정보**:
- 참여자 아바타, 이름, 온라인 상태
- 마지막 메시지 미리보기, 타임스탬프
- 읽지 않은 메시지 수
- 에이전트 참여 여부, E2EE 활성 여부

**대화 필터링**: 전체 / 보안(E2EE) / Agent 포함 / 안 읽음

**그룹 관리**: 그룹 생성 (멤버 검색·선택, 이름·아바타 설정), 멤버 추가/제거, 그룹명 수정, 역할 구분 (owner/member), 그룹 나가기

### 3.3 메시징

**메시지 종류**:
- 내 메시지, 상대방 메시지, 에이전트 메시지, 시스템 메시지 (멤버 입장/퇴장 등)

**메시지 포맷**:
- 일반 텍스트
- Markdown (볼드, 이탤릭, 링크, 코드 블록 with 구문 강조)
- 미디어 첨부 (이미지, 파일. 최대 50MB)
- 리치 블록 — 에이전트가 카드(제목+설명+이미지+버튼), 인라인 액션 버튼을 포함할 수 있음

**전송 상태**: 전송 중 → 전송됨 → 전달됨 → 읽음 (4단계)

**타이핑 인디케이터**: 상대방이 입력 중일 때 표시

**오프라인**: 로컬 캐시로 오프라인 메시지 조회 가능. 전송 대기열 표시

### 3.4 에이전트

Paw의 핵심 차별점입니다.

**에이전트란**: 대화에 참여하여 사용자의 메시지에 실시간으로 응답하는 AI입니다. 사람과 같은 대화 공간에 존재하지만, 에이전트임이 명확히 구별됩니다.

**에이전트 초대 & 동의**:
- 대화에 에이전트를 초대할 때 모든 참여자의 동의가 필요
- 에이전트가 어떤 권한을 가지는지 (메시지 읽기, 응답 전송, 파일 접근 등) 투명하게 표시
- 동의 거부 시 에이전트 추가되지 않음

**에이전트 스트리밍 응답** — Paw의 핵심 경험:
1. 사용자가 메시지를 보내면 에이전트가 실시간으로 응답 생성
2. 텍스트가 토큰 단위로 실시간 추가됨
3. 중간에 도구 호출 발생 시 어떤 도구를 사용 중인지 상태 표시
4. 도구 완료 후 응답 계속
5. 최종 완료 시 소요 시간, 토큰 수 등 메타데이터 표시

**에이전트 마켓플레이스**:
- 에이전트 검색 (이름, 설명), 카테고리 필터링 (생산성 / 코드 / 리서치 / 보안)
- Featured 에이전트 섹션
- 에이전트 정보: 이름, 아이콘, 설명, 카테고리, 사용자 수, 별점/리뷰, 필요 권한 목록
- 원클릭 설치/제거
- 설치 후 원하는 대화에 초대

**사전 등록 에이전트**:
| 에이전트 | 카테고리 | 역할 |
|---------|---------|------|
| Aria | 생산성 | 개인 비서. 미팅 요약, 일정, 컨텍스트 추적 |
| Code Assistant | 코드 | 개발 도우미. 코드 리뷰, 디버깅, 리팩터링 |
| Research Agent | 리서치 | 문서 종합. 여러 소스를 분석하여 요약 |
| Privacy Sentinel | 보안 | 보안 감시. 권한 노출, 개인정보 모니터링 |

### 3.5 E2EE (종단간 암호화)

- 1:1 또는 그룹 대화에서 종단간 암호화 활성화 가능
- 대화에 E2EE 활성/비활성 상태 표시
- 상대방과 숫자 코드 또는 QR 코드로 키 검증
- 에이전트가 E2EE 대화에 참여할 수 있음 (에이전트의 디바이스 키가 세션에 추가)
- 에이전트 제거 시 새 세션 생성 (에이전트 키 제거)

### 3.6 사용자 프로필

**내 프로필**: 아바타 (사진 또는 이니셜), 표시 이름, username (변경 불가), 전화번호 (마스킹), 전화번호 검색 허용 토글

**타인 프로필**: 아바타, 표시 이름, username, 온라인 상태, 메시지 보내기, 차단/해제

### 3.7 검색

- 메시지 전문 검색 (FTS) — 결과에서 해당 대화로 이동
- 대화 이름 검색
- 사용자 검색 (username, 전화번호)

### 3.8 설정

- **알림**: 메시지 알림, 미리보기, 그룹 알림 on/off
- **개인정보**: 읽음 확인, 타이핑 표시, 전화번호 검색 허용
- **보안**: 등록된 디바이스 관리, E2EE 키 관리, 클라우드 백업
- **테마**: 다크/라이트 전환
- **기타**: 도움말, 앱 버전, 로그아웃

### 3.9 브로드캐스트 채널

소유자만 메시지를 보낼 수 있는 일방향 채널. 구독/구독 해제.

### 3.10 멀티 디바이스

같은 계정으로 여러 기기에서 동시 사용. 디바이스별 이름·등록일 관리. 앱 복귀 시 자동 동기화.

### 3.11 미디어 & 파일

이미지, 문서 등 첨부 (최대 50MB). 업로드 진행률 표시. 카메라, 갤러리, 파일 선택 지원.

### 3.12 알림

푸시 알림 (Android/iOS). 대화별 음소거. 알림에서 해당 대화로 딥링크.

### 3.13 클라우드 백업

암호화된 백업 생성/복원. 자동/수동 백업 주기 설정.

### 3.14 차단 & 신고

사용자 차단/해제. 메시지·사용자 신고. 차단 목록 관리.

### 3.15 도움말

에이전트 권한 관리, E2EE 작동 원리, 멀티 디바이스, 백업, 차단 등에 대한 FAQ.

## 4. Design Considerations

1. **에이전트 vs 사람**: 에이전트 메시지는 사람과 명확히 구별되어야 하지만, 대화 흐름에서 자연스러워야 합니다
2. **스트리밍 경험**: 에이전트가 응답을 생성하는 동안의 경험이 Paw의 핵심 차별 요소입니다. 도구 호출 상태도 직관적이어야 합니다
3. **동의와 투명성**: 에이전트의 대화 접근에 대한 사용자 동의가 명확하고 신뢰를 줄 수 있어야 합니다
4. **보안 인지**: E2EE 상태 등 보안 정보가 과하지 않으면서도 필요할 때 접근 가능해야 합니다
5. **오프라인 상태**: 전송 대기, 동기화 상태 등 네트워크 불안정 시의 사용자 경험
6. **한국어 + 영어 혼용 환경**
